How the Rankings Work

Public data, published formulas, every number shown. Here's exactly how we decide who's worst.

What a #1 ranking means

The worst-ranked insurer in a state has the worst composite score on the public regulator metrics available for that state, among insurers with enough volume to measure fairly. It is not a legal finding, an allegation of fraud, or a statement about any individual claim. Every underlying numerator and denominator is displayed on the state pages.

The metrics

MetricFormulaSource
Claims denied in-network claims denied ÷ received CMS Transparency in Coverage PUF (issuer-level, individual-market marketplace plans; ~31 healthcare.gov states)
Confirmed complaints complaints the regulator confirmed ÷ member-months × 100,000 State insurance department complaint data (currently Texas TDI)
Appeals overturned appeals overturned ÷ appeals decided CMS TC-PUF internal appeals; NY DFS external appeals when available
Enforcement actions regulator enforcement actions ÷ member-months × 1,000,000 State enforcement records (currently California DMHC), penalty dollars shown alongside

Population normalization: per-member denominators use member-months from the CMS Medical Loss Ratio public use files (comprehensive commercial coverage: individual, small group, and large group markets) — so a giant insurer isn't punished for having more customers, and a small one isn't hidden by having few.

Zero-filling: complaint and enforcement rates are computed for every insurer in a covered state's insured universe — an insurer absent from the complaint file genuinely drew zero complaints and scores well on that metric. Denial and overturn rates are only measured where CMS reports them; absence means unmeasured, never assumed bad.

Scoring

  1. Each metric is min–max normalized within its state (0 = best observed, 1 = worst observed). Values are never compared across states.
  2. Metrics combine into a weighted composite{% if report.weights %} (current weights: {% for metric, weight in report.weights.items %}{{ metric }} {{ weight }}{% if not forloop.last %}, {% endif %}{% endfor %}){% endif %}, with weights renormalized over the metrics each insurer actually has.
  3. Scores are discounted by metric coverage, so an insurer measurable on one thin metric can't out-rank insurers with bad records across several. Each row's coverage is published in the data.
  4. The result is scaled 0–100 and ranked, worst first, with deterministic tie-breaking.

Reliability thresholds

Small numbers make noisy accusations, so insurers below volume floors are excluded (and listed as excluded in the published data{% if report.thresholds %}; current floors: {% for name, value in report.thresholds.items %}{{ name }} = {{ value }}{% if not forloop.last %}, {% endif %}{% endfor %}{% endif %}). States with fewer than two measurable insurers, or with too little statewide signal on a metric, are reported as having insufficient data rather than force-ranked.

Regulator data lags reality: CMS files are annual (the transparency file reports claims from two plan-years prior), while state complaint and enforcement feeds use a trailing twelve-month window. Each source's vintage is shown above and republished every month.

Known limitations

  • Coverage is uneven: marketplace denial rates exist only for healthcare.gov states, and few states publish machine-readable complaint or enforcement data. States without enough data are labeled, not guessed at.
  • Metrics measure different market segments (marketplace plans vs. the whole commercial book); each insurer's row shows exactly which metrics were measured.
  • Complaint propensity varies by product mix and demographics; confirmed-only counting and within-state comparison reduce but don't eliminate this.

The pipeline, alias tables, thresholds, and full methodology are open source, and the complete machine-readable dataset (including excluded insurers and unmatched-name reports) is published with each monthly run in the worst-insurance-pipeline-by-state repository.
